┌──────────────────────────────────────────────────────────────┐
│  STEP 1: KONFIGURASI XAMPP (DI LAPTOP)                      │
└──────────────────────────────────────────────────────────────┘

  ► Cara Otomatis (RECOMMENDED):
    1. Buka folder: C:\xampp\htdocs\gembira
    2. Klik kanan: enable_network_access.bat
    3. Pilih: "Run as administrator"
    4. Tunggu selesai
    5. Restart Apache di XAMPP Control Panel

  ► Cara Manual:
    1. Buka file:
       C:\xampp\apache\conf\extra\httpd-xampp.conf

    2. Cari baris:
       Require local

    3. Ubah jadi:
       Require all granted

    4. Save file
    5. Restart Apache

┌──────────────────────────────────────────────────────────────┐
│  STEP 2: CEK FIREWALL WINDOWS                                │
└──────────────────────────────────────────────────────────────┘

  ► Option A: Disable Firewall (Sementara)
    1. Buka: Windows Defender Firewall
    2. Click: Turn Windows Defender Firewall on or off
    3. Pilih: Turn off (Private networks only)

  ► Option B: Allow Apache (Lebih Aman)
    Script otomatis sudah menambahkan rule firewall!
    Check: Windows Defender Firewall → Inbound Rules → XAMPP Apache

┌──────────────────────────────────────────────────────────────┐
│  TROUBLESHOOTING                                             │
└──────────────────────────────────────────────────────────────┘

  ❌ "Site can't be reached"
     → HP dan Laptop beda WiFi?
     → Firewall masih block?
     → Apache tidak running?

  ❌ "ERR_CONNECTION_REFUSED"
     → Apache tidak running
     → Restart Apache di XAMPP Control Panel

  ❌ "403 Forbidden"
     → Config belum diubah
     → Jalankan enable_network_access.bat lagi

  ❌ Halaman Blank
     → Clear cache Symfony:
       cd C:\xampp\htdocs\gembira
       php bin/console cache:clear
